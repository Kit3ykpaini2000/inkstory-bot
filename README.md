# inkstory_parser

Telegram-бот для организации литературного конкурса на [inkstory.net](https://inkstory.net).  
Парсит посты участников, распределяет их по жюри и собирает результаты проверки.

## Возможности

- Автоматический сбор ссылок на посты через API inkstory.net
- Парсинг текстов и авторов постов
- Умное распределение постов между жюри (балансировка нагрузки)
- AI-проверка текста через Groq API (орфография, грамматика, пунктуация)
- Планировщик: автозапуск парсера каждые N минут + финальный сбор в 23:55
- Панель администратора: верификация жюри, статистика, логи
- SQLite с WAL-режимом для безопасной параллельной работы

## Установка

```bash
# 1. Клонировать репозиторий
git clone https://github.com/Kit3ykpaini2000/inkstory_parser.git
cd inkstory_parser

# 2. Установить зависимости
pip install -r requirements.txt

# 3. Настроить переменные окружения
cp .env.example .env
# Открыть .env и заполнить BOT_TOKEN и GROQ_API_KEY
```

## Настройка

Скопируй `.env.example` в `.env` и заполни:

```env
BOT_TOKEN=your_bot_token_here
PARSER_INTERVAL=30
GROQ_API_KEY=your_groq_api_key_here
```

- `BOT_TOKEN` — токен бота от [@BotFather](https://t.me/BotFather)
- `PARSER_INTERVAL` — интервал автозапуска парсера в минутах (по умолчанию 30)
- `GROQ_API_KEY` — ключ от [console.groq.com](https://console.groq.com)

## База данных

Перед первым запуском создай БД по схеме из `data/main.db.example`.  
Файл `data/main.db` не хранится в репозитории — создаётся локально.

## Запуск

```bash
# Запустить бота
python bot.py

# Запустить парсер вручную (без бота)
python main.py
```

## Структура проекта

```
inkstory_parser/
├── bot.py                  # Точка входа бота, планировщик задач
├── main.py                 # Точка входа парсера
├── bot/
│   ├── handlers/
│   │   ├── user.py         # Команды жюри (/next, /register, /stats)
│   │   └── admin.py        # Панель администратора
│   └── keyboards.py        # Inline-клавиатуры
├── parser/
│   ├── links.py            # Сбор ссылок через API
│   ├── posts.py            # Парсинг постов
│   └── queue_manager.py    # Управление очередью жюри
├── utils/
│   ├── database.py         # Подключение к SQLite
│   ├── ai_utils.py         # Проверка текста через Groq
│   ├── word_counter.py     # Подсчёт слов
│   └── logger.py           # Логирование
├── scripts/
│   └── migrate.py          # Миграция БД
├── data/
│   └── main.db.example     # Схема базы данных
├── .env.example            # Пример конфигурации
└── requirements.txt
```

## Команды бота

| Команда | Описание |
|---|---|
| `/start` | Приветствие и список команд |
| `/register` | Регистрация жюри |
| `/next` | Получить следующий пост для проверки |
| `/stats` | Моя статистика |
| `/fullstats` | Общая статистика конкурса |
| `/admin` | Панель администратора |
